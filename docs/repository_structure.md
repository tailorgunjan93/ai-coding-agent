# Repository Structure

## Overview
This document explains the purpose of each folder and file in the project.  
It ensures contributors understand where to add code, tests, documentation, and CI/CD configurations.

---

## Root Files
- **README.md** → Project overview, setup instructions, features, and usage.
- **planning.md** → Auto-generated plan for each enhancement or app, requiring human approval.
- **LICENSE** → Open-source license (MIT).
- **.gitignore** → Defines files/folders excluded from version control.

---

## Directories

### `/src`
- **main/agent/nodes/** → Implementation of LangGraph nodes:
  - Planning, HumanApproval, Model, Lifecycle, Tester, DB, Git, CI/CD, DesignPattern, Security, Output.
- **main/utils/** → Helper modules:
  - Code commenter, design pattern enforcer, DB handler, Git helper, CI/CD generator, test runner.
- **extension/** → VS Code extension files:
  - Commands, UI components, manifest.json.
- **langgraph_schema.json** → JSON schema of the agent workflow.

---

### `/tests`
- **unit/** → Unit tests for individual functions/classes.
- **integration/** → Tests for workflows across modules.
- **edge_cases/** → Stress and boundary condition tests.
- **db_tests/** → Database-specific tests.
- **coverage/** → Coverage reports generated by CI/CD.

---

### `/agent-graphs`
- **code_graph.json** → Structural diagram of code modules.
- **data_flow_graph.json** → Runtime flow diagram.
- **visualizations/** → Rendered diagrams for documentation.

---

### `/ci`
- **github-actions.yml** → GitHub Actions pipeline.
- **gitlab-ci.yml** → GitLab CI/CD pipeline.
- **azure-pipelines.yml** → Azure DevOps pipeline.
- Ensures portability across major CI/CD platforms.

---

### `/docs`
- **architecture.md** → LangGraph schema explained visually and textually.
- **workflow.md** → End-to-end lifecycle description.
- **design_patterns.md** → SOLID principles and enforced design patterns.
- **testing_strategy.md** → Continuous testing and correction loop.
- **security_node.md** → Linting, static analysis, vulnerability scans.
- **ci_cd_integration.md** → Unified CI/CD strategy across GitHub, GitLab, Azure.

---

### `/db`
- **schema.sql** → Database schema definition.
- **migrations/** → Migration scripts for schema evolution.
- **orm_models/** → ORM classes for DB interaction.

---

### `/.vscode`
- **settings.json** → VS Code workspace settings.
- **launch.json** → Debugging configurations.
- **tasks.json** → Build/test tasks.

---

### `/.github`
- **ISSUE_TEMPLATE.md** → Template for bug reports/features.
- **PULL_REQUEST_TEMPLATE.md** → Template for PRs.
- **workflows/** → Additional GitHub workflows.

---

## Recommendations
- Keep code modular and documented.
- Link commits to `planning.md` for traceability.
- Store test coverage and graphs for review.
- Treat CI/CD configs as **non-negotiable quality gates**.
