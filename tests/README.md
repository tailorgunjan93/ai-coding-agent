# Tests

This directory contains all tests for the project.

## Instructions

1.  **Mandatory Tests**: Every new function, node, or wrapper must have a corresponding unit test.
2.  **Use Pytest**: We use `pytest` as our testing framework.
3.  **Mocking**: Use `unittest.mock` or `pytest-mock` to mock external dependencies (wrappers, APIs) when testing nodes.
4.  **Directory Structure**:
    *   `tests/unit/`: Unit tests for individual components.
    *   `tests/integration/`: Tests for multiple components working together.

## Tasks

- [ ] Write unit tests for every wrapper and node.
- [ ] Use `pytest` to run tests.
- [ ] Ensure 100% coverage for new logic.
